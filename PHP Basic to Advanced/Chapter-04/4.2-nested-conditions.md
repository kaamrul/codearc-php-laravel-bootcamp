# অধ্যায় ৪: পিএইচপি কন্ট্রোল স্ট্রাকচার (PHP Control Structure)

## ৪.২ নেস্টেড কন্ডিশন (Nested Conditions)

PHP তে একটি `if` statement এর ভিতরে আরেকটি `if` statement ব্যবহার করাকে **Nested Condition** বলা হয়।

যখন একটি condition সত্য হওয়ার পরে আরও একটি condition যাচাই করার প্রয়োজন হয়, তখন Nested Condition ব্যবহার করা হয়। এটি complex decision making system তৈরিতে অত্যন্ত গুরুত্বপূর্ণ।


### Nested if Statement
---

Nested condition ব্যবহার করে একাধিক ধাপে condition যাচাই করা যায়।

### Syntax

```php
if (condition1) {

    if (condition2) {
        // condition1 এবং condition2 উভয়ই সত্য হলে এই কোড চলবে
    }

}
```

### Example

```php
<?php
    $age = 25;
    $hasNID = true;

    if ($age >= 18) {
        if ($hasNID == true) {
            echo "আপনি ভোট দিতে পারবেন";
        }
    }
?>
```

### Output

```text
আপনি ভোট দিতে পারবেন
```


### Nested if...else Statement
---

Nested condition এর সাথে `else` ব্যবহার করে বিভিন্ন পরিস্থিতি handle করা যায়।

### Example

```php
<?php
    $age = 16;
    $hasNID = false;

    if ($age >= 18) {
        if ($hasNID == true) {
            echo "আপনি ভোট দিতে পারবেন";
        } else {
            echo "আপনার NID নেই";
        }
    } else {
        echo "আপনি এখনও ভোট দেওয়ার যোগ্য নন";
    }
?>
```

### Output

```text
আপনি এখনও ভোট দেওয়ার যোগ্য নন
```

### Real-Life Example: Role Permission System
---

Nested Conditions বাস্তব জীবনের Role Permission System তৈরিতে অনেক বেশি ব্যবহার করা হয়।

যেমন:
- একজন User login করেছে কিনা যাচাই করা
- User Admin কিনা যাচাই করা
- Admin হলে Dashboard access দেওয়া

### Example

```php
<?php
    $isLoggedIn = true;
    $userRole = "admin";

    if ($isLoggedIn == true) {
        if ($userRole == "admin") {
            echo "Welcome Admin";
        } else {
            echo "Access Denied";
        }
    } else {
        echo "Please Login First";
    }
?>
```

### Output

```text
Welcome Admin
```

### Explanation

উপরের উদাহরণে প্রথমে যাচাই করা হয়েছে User login করেছে কিনা।

যদি User login করা থাকে, তাহলে দ্বিতীয় condition এর মাধ্যমে যাচাই করা হয়েছে User এর role `admin` কিনা।

- User যদি admin হয় → `Welcome Admin` দেখাবে
- User login করা থাকলেও admin না হলে → `Access Denied` দেখাবে
- User login না করলে → `Please Login First` দেখাবে

এভাবেই Nested Condition ব্যবহার করে multi-level permission system তৈরি করা যায়।


## 🎯 কেন Nested Conditions গুরুত্বপূর্ণ?

Nested Condition ব্যবহার করে:

- Complex logic তৈরি করা যায়
- একাধিক condition একসাথে যাচাই করা যায়
- Decision making আরও শক্তিশালী হয়
- Real-world application logic implement করা যায়


## ⚠️ গুরুত্বপূর্ণ বিষয়

অতিরিক্ত Nested Condition ব্যবহার করলে কোড জটিল হয়ে যেতে পারে। তাই প্রয়োজন ছাড়া অনেক বেশি nesting ব্যবহার না করাই ভালো।
