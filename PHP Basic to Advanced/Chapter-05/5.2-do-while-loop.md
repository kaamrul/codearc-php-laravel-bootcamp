# অধ্যায় ৫: লুপ এবং ইটারেসন (PHP Loops & Iteration)

## ৫.২ Do While Loop

PHP তে `do while` loop ব্যবহার করা হয় এমন কোনো কোড ব্লককে অন্তত একবার রান করানোর জন্য, এরপর একটি নির্দিষ্ট শর্ত (Condition) সত্য থাকা পর্যন্ত সেটিকে বারবার এক্সিকিউট করার জন্য।

অনেক সময় এমন পরিস্থিতি আসে যেখানে আমরা চাই কোনো কাজ প্রথমে একবার অবশ্যই সম্পন্ন হোক, তারপর শর্ত অনুযায়ী সেটি আবার চলবে কি না তা নির্ধারণ করা হবে। এই ধরনের পরিস্থিতিতে `do while` loop সবচেয়ে উপযোগী।


### Do While Statement
---

`do while` loop প্রথমে কোড ব্লকটি একবার রান করে, তারপর কন্ডিশন বা শর্ত যাচাই করে।

যদি কন্ডিশন সত্য (True) হয়, তাহলে আবার কোড রান হবে। প্রতিবার কোড রান করার পর পুনরায় কন্ডিশন চেক করা হবে। যতক্ষণ পর্যন্ত শর্তটি সত্য থাকবে, লুপটি চলতে থাকবে। শর্ত মিথ্যা (False) হয়ে গেলে লুপ বন্ধ হয়ে যাবে।

সবচেয়ে গুরুত্বপূর্ণ বিষয় হলো — শর্ত শুরুতেই মিথ্যা হলেও `do while` loop অন্তত একবার চলবেই।

---

### While Loop এবং Do While Loop এর পার্থক্য

| While Loop | Do While Loop |
|---|---|
| আগে condition check করে | আগে code execute করে |
| condition false হলে একবারও চলে না | অন্তত একবার চলবেই |
| Entry-controlled loop | Exit-controlled loop |

---

### Syntax

```php
<?php

initialization; // লুপের শুরুর মান নির্ধারণ

do {

    // code to be executed;

    increment/decrement; // মান পরিবর্তন

} while (condition);

?>
```


### Example 1: 

আসুন একটি বাস্তব উদাহরণ দেখি। মনে করুন, একটি মোবাইল অ্যাপে লগইন করার পর সিস্টেম অন্তত একবার ইউজারের তথ্য যাচাই করবে। এরপর যদি তথ্য সঠিক না হয়, তখন আবার চেষ্টা করতে দিবে।

```php

<?php

    $attempt = 1;

    do {

        echo "লগইন তথ্য যাচাই করা হচ্ছে... চেষ্টা নম্বর: $attempt <br>";

        $attempt++;

    } while ($attempt <= 3);

?>

```


### এক্সাম্পল ডিটেইলস ও যেভাবে কাজ করবে:
- **শুরুর ধাপ:** প্রথমে `$attempt` এর মান ১ সেট করা হয়েছে
- **কোড রান:** `do` ব্লকের ভেতরের কোড প্রথমে একবার রান হবে
- **মান পরিবর্তন:** `$attempt++` এর মাধ্যমে মান ১ করে বাড়বে
- **শর্ত যাচাই:** এরপর চেক করবে `$attempt <= 3` কি সত্য?
- **পুনরাবৃত্তি:** শর্ত সত্য হলে আবার কোড রান করবে
- **লুপ সমাপ্তি:** যখন `$attempt` এর মান ৪ হবে, তখন শর্ত মিথ্যা হয়ে যাবে এবং loop বন্ধ হবে

### Output

```text
লগইন তথ্য যাচাই করা হচ্ছে... চেষ্টা নম্বর: 1
লগইন তথ্য যাচাই করা হচ্ছে... চেষ্টা নম্বর: 2
লগইন তথ্য যাচাই করা হচ্ছে... চেষ্টা নম্বর: 3
```


### Example 2: পেমেন্ট প্রসেস রিট্রাই সিস্টেম (Payment Process Retry System)

বাস্তব জীবনের প্রজেক্টে যখন আমরা bKash, SSLCommerz বা Stripe-এর মতো পেমেন্ট গেটওয়ে ব্যবহার করি, তখন মাঝে মাঝে নেটওয়ার্ক সমস্যার কারণে পেমেন্ট রিকোয়েস্ট প্রথমবারে সফল হয় না। এই ধরনের পরিস্থিতিতে, আমরা ইউজারকে সরাসরি "Payment Failed" না দেখিয়ে ব্যাকএন্ড থেকে অন্তত একবার চেষ্টা করি এবং ফেইল হলে আরও ২/৩ বার রিট্রাই করার লজিক তৈরি করি।


যেহেতু পেমেন্ট প্রসেসটি শুরুতে অন্তত একবার চালাতেই হবে, তাই এখানে `do while loop` ব্যবহার করা সবচেয়ে বুদ্ধিমানের কাজ।

```php
<?php

    $paymentSuccess = false; // মনে করি পেমেন্ট এখনো সফল হয়নি
    $currentAttempt = 1;     // প্রথম চেষ্টা
    $maxAttempts = 3;        // সর্বোচ্চ ৩ বার চেষ্টা করা হবে

    do {
        echo "পেমেন্ট গেটওয়ের সাথে যোগাযোগ করা হচ্ছে... চেষ্টা নম্বর: $currentAttempt <br>";

        // বাস্তব প্রজেক্টে এখানে bKash বা Stripe API রান হতো। 
        // আমরা বোঝার সুবিধার্থে ৩ নম্বর চেষ্টায় পেমেন্ট সফল (True) করে দিচ্ছি:
        if ($currentAttempt == 3) {
            $paymentSuccess = true;
        }

        // যদি পেমেন্ট সফল হয়ে যায়, তবে লুপ থেকে বের হওয়ার জন্য মেসেজ দেখাব
        if ($paymentSuccess) {
            echo "<strong>পেমেন্ট সফল হয়েছে! আপনার অর্ডারটি নিশ্চিত করা হলো।</strong><br>";
            break; // পেমেন্ট হয়ে গেলে লুপটি এখানেই জোরপূর্বক বন্ধ করে দেওয়া হলো
        } else {
            echo "পেমেন্ট ব্যর্থ হয়েছে। আবার চেষ্টা করা হচ্ছে...<br><br>";
        }

        $currentAttempt++; // চেষ্টার সংখ্যা ১ করে বাড়ছে

    } while ($currentAttempt <= $maxAttempts); // condition: যতক্ষণ চেষ্টার সংখ্যা ৩ বা তার কম থাকবে

    // যদি ৩ বার চেষ্টা করার পরও পেমেন্ট সফল না হয়
    if (!$paymentSuccess) {
        echo "<br>দুঃখিত, নেটওয়ার্ক সমস্যার কারণে পেমেন্ট করা যায়নি। দয়া করে কিছুক্ষণ পর আবার চেষ্টা করুন।";
    }

?>
```

### এক্সাম্পল ডিটেইলস ও যেভাবে কাজ করবে:
- **বাধ্যতামূলক প্রথম রান:** লুপটি শুরুতে কোনো কন্ডিশন চেক না করেই `$currentAttempt = 1` এর জন্য পেমেন্ট রিকোয়েস্ট পাঠাবে।
- **`break` স্টেটমেন্টের কাজ:** লুপের মাঝে যদি পেমেন্ট সফল (`$paymentSuccess = true`) হয়ে যায়, তবে লুপটি আর অযথা না ঘুরিয়ে `break` দিয়ে তখনই বন্ধ করে দেওয়া হয়।
- **ধাপে ধাপে কার্যপ্রবাহ:**
    - **১ম চেষ্টা:** পেমেন্ট ফেইল হবে। লুপের মান বেড়ে হবে ২। শর্ত চেক হবে `(২ <= ৩)` যা সত্য।
    - **২য় চেষ্টা:** পেমেন্ট আবার ফেইল হবে। লুপের মান বেড়ে হবে ৩। শর্ত চেক হবে `(৩ <= ৩)` যা সত্য।
    - **৩য় চেষ্টা:** এবার আমাদের লজিক অনুযায়ী পেমেন্ট সফল হবে এবং `break` এর কারণে লুপটি বন্ধ হয়ে যাবে।



### Output

```text

পেমেন্ট ব্যর্থ হয়েছে। আবার চেষ্টা করা হচ্ছে...

পেমেন্ট ব্যর্থ হয়েছে। আবার চেষ্টা করা হচ্ছে...

পেমেন্ট গেটওয়ের সাথে যোগাযোগ করা হচ্ছে... চেষ্টা নম্বর: 3

```

### ⚠️ সতর্কতা (Infinite Loop):

লুপের ভেতরে যদি মান পরিবর্তনের ধাপ বা `increment/decrement` করতে ভুলে যান, তবে কন্ডিশনটি আজীবনের জন্য সত্য রয়ে যাবে। এর ফলে লুপটি কখনো বন্ধ হবে না, যাকে Infinite Loop বলা হয়। এটি আপনার ব্রাউজার বা সার্ভার ক্র্যাশ করাতে পারে।

```php
<?php

    $i = 1;

    do {

        echo $i;

    } while ($i <= 5);

?>
```

> এখানে $i এর মান কখনো পরিবর্তন হচ্ছে না, তাই condition সবসময় সত্য থাকবে এবং loop অনন্তকাল চলবে।



### 🎯 কেন Do While Loop গুরুত্বপূর্ণ?

এটি ব্যবহার করে:

- কোনো কাজ অন্তত একবার চালানো নিশ্চিত করা যায়
- Login System তৈরি করা যায়
- Menu-driven application তৈরি করা যায়
- Game retry system তৈরি করা যায়
- User input validation করা যায়




### 📚 সারসংক্ষেপ

> `do while loop` হলো পিএইচপির একটি Exit-controlled Loop, যেখানে প্রথমে কোড execute হয় এবং পরে condition check করা হয়। তাই condition শুরুতেই false হলেও loop অন্তত একবার চলবেই।
